01 – Overview

The big idea

You can now type a plain-English request in an AI tool, and it will reach into your Figma file and build or edit designs using your actual components and design system – not generic shapes.

Figma's canvas is now open to AI agents. This means an AI tool like Claude can connect directly to your Figma file, read your component library, understand your design system, and create or modify designs on your behalf – all from a simple text prompt.

This isn't a plugin that generates Lorem Ipsum frames. The agent uses your real components, your real variables, and your real naming conventions. The quality of the output is directly tied to the quality of your design system.

02 – Glossary

Key terms explained

Before diving into setup, here are the concepts you'll encounter and what they actually mean.

AI Agent
An AI tool that can take actions on your behalf – not just answer questions, but actually open files, create designs, run checks, and iterate. Claude Code, Cursor, Codex, Augment, and Warp are all agents.
MCP Server
Model Context Protocol. A standardised way for AI tools to connect to external apps and services. Figma runs an MCP server – when an AI agent connects to it, the agent gains the ability to read and write your Figma files directly. Think of it as a secure API bridge between the AI and Figma.
Claude Code
Anthropic's AI agent. It runs in your terminal (command line), not in a browser window. You type prompts, it carries out tasks – including connecting to Figma via the MCP server.
use_figma
The core tool the agent uses once connected to Figma. It empowers the agent to operate on your Figma canvas using your design system – selecting components, placing frames, applying variables, and building layouts.
generate_figma_design
A companion tool that translates HTML from live apps or websites into editable Figma layers. Useful for reverse-engineering existing interfaces into your design system.
Skills
Markdown files that tell the agent how to work in your Figma file. They define the workflow: which steps to take, what sequencing to follow, which conventions to use. Skills encode your team's design decisions so every agent run follows your rules – not its own defaults.
Code Connect
A Figma feature (set up by a developer) that links each Figma component to its real code counterpart in the codebase. With Code Connect, the agent doesn't just see a "Button" – it sees the actual <Button> React component. This enables the agent to generate real, production-ready code.
Personal Access Token
A private key generated in your Figma account settings. It's how Claude authenticates with Figma on your behalf. Treat it like a password – never share it or commit it to a document.
CLAUDE.md
A configuration file you can place inside a project folder. Rules saved here are automatically loaded every session when you're working in that folder. Good for project-specific defaults like "always use this component library" or "always use 375×848 iPhone frames".
03 – Requirements

What You Need

Three things are required before the workflow can run. Two of them are accounts you likely already have; one is a small piece of software to install.

🎨

A Figma account with your design system

Your components, variables, and styles should already be set up in Figma. The better organised your component library, the better the agent's output.

🤖

An AI coding agent

This guide covers Claude Code. Other supported agents include Cursor, Codex, Augment, and Warp – all work with the same Figma MCP server.

🔌

Figma MCP server connection

A one-time configuration that connects your chosen AI agent to Figma. This is what allows the agent to read and write your Figma files. No coding required.

04 – Approvals

Permissions & stakeholders

This workflow involves a few external systems, so a handful of teams may need to be looped in before you can get started. Here's a map of who owns what.

1

Install Claude Code on your laptop

Claude Code is a command-line tool you install locally. It doesn't have a graphical interface – you type prompts in a terminal. Your IT team needs to permit the installation, and the tool needs to be able to make outbound HTTPS calls to Anthropic's API.

IT Team
2

Connect Claude Code to Figma via OAuth

A single terminal command registers Figma's MCP server with Claude Code. You then run /mcp inside Claude Code, select Figma, and choose Authenticate – a browser window opens, you log into your Figma account and approve access. No tokens, no config files. Claude Code manages the connection from there on.

You
3

Data privacy check

When the agent reads your Figma file to understand your components, that design content is sent to Anthropic's API as context. Worth flagging to your security or legal team so they can check Anthropic's data processing terms against your company's data classification policy.

05 – CLI setup

Terminal installation walkthrough

These are the full terminal commands to get Claude Code installed on a Mac and connected to Figma. Follow each step in order. If your IT team handles the installation, share this section with them.

Mac only. These instructions are for macOS using the ZSH shell. If you're on Windows or Linux, the commands differ slightly – refer to the official Claude Code documentation.
1

Install Homebrew (Mac package manager)

Homebrew is a tool that makes it easy to install software on a Mac. If you already have it, skip to step 2.

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Once installed, add it to your shell's path so it's available in every terminal session:

echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zshrc
source ~/.zshrc

Verify it worked:

brew --version
2

Install NVM (Node version manager)

NVM lets you install and manage versions of Node.js, which Claude Code requires.

brew install nvm

Add NVM to your shell:

mkdir ~/.nvm
echo 'export NVM_DIR="$HOME/.nvm"' >> ~/.zshrc
echo '[ -s "/opt/homebrew/opt/nvm/nvm.sh" ] && \. "/opt/homebrew/opt/nvm/nvm.sh"' >> ~/.zshrc
source ~/.zshrc

Verify:

nvm --version
3

Install Node.js

Install the latest long-term support (LTS) version of Node.js:

nvm install --lts
nvm use --lts

Verify both Node and npm are installed:

node --version
npm --version
4

Install Claude Code

Install Claude Code globally using npm:

npm install -g @anthropic-ai/claude-code

Verify the installation:

claude --version
5

Log into Claude Code

Start Claude Code and follow the login prompts (it will open a browser window to authenticate with your Anthropic account):

claude
6

Connect Claude Code to Figma's MCP server

Register Figma's remote MCP server with Claude Code. The --scope user flag makes it available across all your projects, not just the current folder:

claude mcp add --scope user --transport http figma https://mcp.figma.com/mcp

Then verify it was added:

claude mcp list
7

Authenticate with Figma via OAuth

No personal access token is needed. Figma uses OAuth – start Claude Code and run the MCP authentication flow:

claude

Inside Claude Code, type /mcp, select figma from the list, then select Authenticate. A browser window will open with Figma's OAuth consent screen. Click Allow access. Once authorised, the browser will confirm the connection and Claude Code will show "Authentication successful. Connected to figma."

8

Test the connection

Create a project folder, navigate into it, and start Claude. Then send a test prompt using a link to a Figma file you have access to:

mkdir ~/my-figma-project
cd ~/my-figma-project
claude

Then type a prompt like:

Look at the Components page in [your Figma link] and list all the components you can see, grouped by category. Don't create anything yet.
Good first test. Starting with "list what you see" before asking it to create anything is a reliable way to confirm the connection is working and the agent can actually read your component library.
06 – Claude app setup

Claude app installation walkthrough

If you're using the Claude desktop app (rather than Claude Code in the terminal), you can connect Figma through the app's settings – no terminal required.

1

Open connector settings

Go to Settings → Connectors → Add custom connector.

2

Enter Figma's MCP endpoint

Enter https://mcp.figma.com/mcp as the URL and give the connector a name.

3

Authenticate via OAuth

An OAuth window will open in your browser – log in to Figma and approve the permissions.

Note: The Figma MCP connection is account-level – saved to your Claude account, not to a specific project folder. Once connected, it's available across all your Claude sessions.
07 – Your Role

Your role as a designer

The rise of AI coding agents doesn't change what makes a great designer, but it fundamentally changes how you work. You are shifting from a builder of static layouts to a director of a living system.

From Manual Labor to Systems Orchestration

Previously, much of your time was spent manually drawing frames and pushing pixels to mimic components. With AI agents, the agent handles the assembly. Your value now lies in Systems Thinking:

  • Defining the Logic: You aren't just placing a button; you are defining the rules for when and how that button appears.
  • Quality Control: The agent is only as good as your Design System. If your naming conventions are messy or your Auto Layout is broken, the agent will fail. Your job is to maintain the "source of truth."
  • Creative Direction: You move from "How do I draw this?" to "Is this the right flow for the user?"

Bridging the Gap: Design is the Code

The biggest shift is in the handoff. Traditionally, developers had to "interpret" your static Figma files, often leading to "pixel-polishing" meetings.

  • Less Interpretation: Because the agent uses Code Connect, it sees your Figma components as actual React elements (or your specific production code). It isn't guessing your intent; it is using the real thing.
  • Functionality-First Development: Since the UI structure is generated directly from your design rules, developers spend less time recreating your visuals and more time on complex logic, security, and performance.

The Reality Check

You still need your core UX skills – empathy, hierarchy, and logical flow. The tool just removes the friction of manual construction for both the designer and front-end developer. You are no longer just a figma user; you are an architect who uses the Figma canvas as a command center for production.

08 – Design to code

Code Connect: design to production components

Code Connect is a Figma feature that links each component in your design system to its actual code counterpart in the codebase. It's an optional layer – relevant when the goal is to produce working code from your designs, not just Figma frames.

What it does

Without Code Connect, when an agent reads a "Button" component in your Figma file, it generates whatever React code it thinks makes sense – probably something generic. With Code Connect, it sees the mapping: "this Figma Button corresponds to the real <Button> React component" – and generates code that calls your actual component with the correct props.

The result is a working, clickable frontend – actual runnable code, not a prototype.

How the mapping works

A developer writes small configuration files – one per component – that declare the link between a Figma component's variants and the corresponding code props. A simplified example for a Button:

Figma component: "Button"
  variant "Type" = "Primary"  →  React prop: variant="primary"
  variant "Size" = "Large"    →  React prop: size="lg"
  text layer "Label"          →  React prop: children

What you need already in place

Code Connect only works if your Figma components already have a corresponding React component library in code. There are two realistic starting points:

You have an existing React component library that mirrors your Figma components – a developer sets up Code Connect, and you're ready to generate code from your designs.

You're starting from scratch with no codebase yet – you design in Figma first, ask the agent to generate a React component library from your Figma components, and then set up Code Connect for future use. This works, but it's a bigger initial investment.

What you control vs. what a developer controls

You own the Figma side entirely: the designs, the component structure, the variants, the naming. The quality of what Code Connect can map directly reflects how well-organised your Figma components are. Inconsistent naming, missing variants, or components that don't cleanly correspond to how developers have built things will create gaps in the mapping.

09 – Prompting

Prompting guide

The quality of the agent's output is directly influenced by how clearly you write your prompts. Here's a template that works well for screen creation tasks.

Task: Create a [screen name] screen in this Figma page: [link here]   Component library: Only use components from this page: [link here]. // If a component you need does not exist there, tell me before proceeding. // Do not pull from other libraries.   Reference design: [link here, or "none"]   Screen spec: Device: iPhone (375×848) Visual tone: [light / dark / branded] Content sections: [e.g. header, form, CTA, footer] Components needed: [e.g. Text Input, Button, Standalone Link]   Rules: - Use vertical auto layout, 40px section gap, 24px horizontal padding - Use section frames with fixed counter width - Use the real iPhone status bar component - All text must use correct text styles from the library - Ask me before making any assumptions about layout or content

Persisting rules with CLAUDE.md

For rules that apply to every session in a project, you can save them in a CLAUDE.md file inside your project folder. Rules saved here are automatically loaded every time you start a session in that folder – so you don't need to repeat yourself in every prompt.

Good candidates for CLAUDE.md rules: which component library to always use, your standard frame dimensions, restrictions on pulling from other libraries, and any conventions specific to the project.

10 – Security

Security

Working with AI agents introduces real security and privacy risks. The speed and convenience of the workflow do not change that. The rules in this section are not guidelines – they are requirements.

GDPR: never expose personal data

The agent reads your Figma file. Everything visible in that file – layer names, content, annotations, placeholder text – is sent to the model as part of the interaction. Any personal data present in the file at that point is a GDPR violation.

Do not work in any file that contains real client data, real user data, or identifiable information about coworkers. This includes names, email addresses, profile photos, account numbers, and any content derived from real people. Only generic, fictional, or fully anonymised placeholder content is acceptable. If you are unsure whether something counts as personal data, treat it as if it does.

This rule is absolute. There are no exceptions based on file visibility settings, team permissions, or how sensitive you think the data is. If personal data is in the file, the agent cannot be used on it.

Work in a sandbox – never in production

Every agent session must take place in a dedicated sandbox: a separate Figma project and file created specifically for AI-assisted work. This must be set up before you run any session.

The design system file is off limits. Production files are off limits. The agent must never have access to either. Once the agent has produced output you are happy with, transfer only the approved elements manually – after review – into the appropriate file. Never let the agent write directly into any shared or live file.

One misrun session in a production file can corrupt components, break variable connections, and overwrite work that affects the entire team. The sandbox rule exists to make sure that never happens.

Restrict what the agent can access on your machine

Claude Code runs via the terminal and has access to your local file system during a session. Do not configure it to run automatically in the background. Do not grant it access beyond what is needed for the specific task you are running. When the session is finished, close it. Do not leave an active session running unattended, and do not connect it to services or directories that have nothing to do with the current task.

Review before you transfer anything

Agent output is a starting point, not a finished result. Nothing produced by the agent should move to a shared library, a production file, or a developer handoff without being reviewed first. Check every output against the brief, the design system, and accessibility standards. The review step is not optional – it is the point at which you take responsibility for what the agent has produced.
